# Context Optimizer — OpenClaw 上下文优化技能

长会话中自动精简上下文，保住关键信息不被压缩掉。

## 内容优先级

|级别|内容类型|策略|
|---|---|---|
|P0 必须保留|核心指令、当前进行中的任务、SOUL/USER 配置|不压缩|
|P1 尽量保留|已完成子任务结果、技术决策、偏好设置|摘要为 1-2 句|
|P2 可以压缩|调试日志、工具中间结果、已完成的旧任务|压缩或删除|
|P3 可以删除|确认回复（"好的""知道了"）、重复提示文本|直接删除|

## 触发条件

|条件|阈值|
|---|---|
|上下文 token 占用率|超过最大上限的 70%|
|会话持续时长|超过 30 分钟|
|手动触发|用户说"压缩上下文"|

## 压缩步骤

```
1. 统计 P0-P3 各占多少 token
2. 删除全部 P3 内容
3. 将 P2 内容压缩为一句摘要
4. 合并 P1 中重复的部分
5. P0 内容不动
6. 输出压缩报告: "释放了 X token（Y -> Z）"
```

## 压缩报告示例

```
压缩前: 45200 token
压缩后: 28100 token (释放 37%)
P0 指令: 保留 | P1 摘要: 保留 | P2 日志: 压缩 | P3 确认: 删除
```
