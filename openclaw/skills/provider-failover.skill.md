# Provider Failover — OpenClaw API 故障切换技能

自动检测 API 提供商是否可用，不可用时无缝切换到备用服务。

## 故障检测

|异常类型|判断条件|
|---|---|
|HTTP 5xx|状态码 >= 500|
|超时|超过 15 秒无响应|
|余额不足|total_balance < 0.01|
|429 限流|状态码 = 429|
|401 鉴权失败|状态码 = 401|

## 切换流程

```
请求主提供商
  -> 成功 -> 正常返回
  -> 失败 -> 等待 5 秒 -> 重试 1 次
       -> 成功 -> 正常返回
       -> 再失败 -> 切换到备用提供商
            -> 成功 -> 标记为主力继续使用
            -> 全部失败 -> 通知用户所有提供商不可用
```

## 提供商优先级

1. 免费 DeepSeek Key（余额最高的优先）
2. 主人付费 DeepSeek Key（兜底）
3. Gemini / Grok（仅当 DeepSeek 全挂时）

## 恢复检查

每 30 分钟自动检查主提供商是否恢复，恢复后自动切回。
切换记录写入日志: logs/provider-failover.log
