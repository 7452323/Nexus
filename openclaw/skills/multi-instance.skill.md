# Multi Instance — OpenClaw 多实例管理技能

在同一台机器上管理多个 OpenClaw 配置实例。

## 实例目录结构

```
~/.openclaw/instances/
├── default/    # 默认实例（日常使用）
├── dev/        # 开发实例（测试新功能）
└── prod/       # 生产实例（稳定运行）
```

每个实例包含完整的 openclaw.json 和 agents/ 子目录。

## 端口分配

|实例|Gateway 端口|
|---|---|
|default|18789|
|dev|18790|
|prod|18791|

## 命令参考

|你说|效果|
|---|---|
|"切换实例 dev"|切换到开发实例（需重启 gateway）|
|"创建实例 test"|复制 default 配置创建新实例|
|"删除实例 test"|删除 test 实例（需用户确认）|
|"列出实例"|显示所有实例及各自状态|

## 检查清单

切换前：
- [ ] 目标实例配置文件是否存在
- [ ] 目标端口是否被占用
- [ ] 是否有正在进行中的任务

创建时：
- [ ] 实例名是否已存在
- [ ] 端口是否已被其他实例使用
